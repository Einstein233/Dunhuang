# 敦煌项目当前说明

## 项目定位

本项目围绕四条主线展开：

1. 气象数据采集与入库
2. 气象数据库查询与智能问答
3. 等效实验方案计算
4. 前端大屏与后台页面展示

现在项目已经不是“从零开始”的状态，而是已经跑通了基础闭环，进入了结构整理和交付前优化阶段。

## 当前已经完成的内容

### 1. 核心系统已经联通

目前已经有以下主要模块：

- `back_con`：Node 后端，负责接口、数据查询、等效服务调用
- `front_con`：后台管理前端
- `WeatherVisualization`：大屏展示前端
- `services/sql-agent`：SQL 智能体服务
- `services/equivalence-service`：等效计算服务
- `services/mysql`：数据库初始化文件与数据维护脚本

也就是说，前端、后端、数据库、智能体、等效服务五块已经具备基础运行能力。

### 2. 数据库主结构已经统一

当前数据库围绕下面三张核心表工作：

- `station_info`：地区与站点信息
- `weather_data`：小时级气象事实数据
- `weather_directory`：数据目录与覆盖范围

早期零散表结构已经不再作为主结构继续扩展。

### 3. 数据导入链路已经可用

目前已经可以：

- 根据地区和经纬度从外部气象接口拉取数据
- 将原始返回整理成系统当前使用的字段结构
- 导入 `weather_data`
- 同步 `station_info` 与 `weather_directory`
- 导出数据库 SQL 快照，便于团队同步

数据库初始化与全量数据快照现在集中在：

- `services/mysql/init/vue_admin.sql`
- `services/mysql/init/weather_data.sql`

### 4. 等效方案闭环已经打通

目前已经支持下面这条链路：

1. 前端选择地区
2. 前端选择自然气候时间窗口
3. 前端选择实验类型
4. 前端选择目标实验室时长
5. 后端查询数据库中的气象数据
6. 后端调用等效服务
7. 前端以表格形式展示结果

这意味着“数据查询 -> 等效计算 -> 结果展示”的主流程已经可以跑通。

### 5. 服务架构正在向微服务方式收口

当前已整理为统一的服务目录：

- `services/mysql`
- `services/sql-agent`
- `services/equivalence-service`

并通过统一的 `services/docker-compose.yml` 管理三项服务，便于整体启动，也便于单独启动其中某一项服务。

## 当前还没有完成的重点

### 1. 自动化数据补充机制还未最终完成

虽然现在已经可以手动拉取和导入数据，但还没有形成完整的生产级自动补数流程。后续还需要完善：

- 定时任务
- 断点续传
- 失败重试
- 多地区批量补数
- 每日额度控制
- 数据质量校验

### 2. SQL 智能体还需要继续优化

当前智能体已经具备基础查询能力，但还需要继续提升：

- 对最新数据库结构的稳定理解
- 工具调用约束
- SQL 安全性
- 结果解释质量
- 面向真实业务问题的可用性

### 3. 等效算法还需要支持更灵活的时间窗口

现在闭环虽然已经可跑，但后续重点不是只做“按整年计算”，而是要支持：

- 用户自定义任意时间窗口
- 系统按年份切分窗口
- 每年分别计算等效结果
- 最终拼接成年份连续的总方案

这是后续等效服务优化的关键方向。

### 4. 交付材料还要继续整理

后续还需要补充更清晰的：

- 部署说明
- 演示流程
- 数据结构说明
- 模块职责说明
- 对照项目协议书的完成情况总结

## 目前可以怎样理解项目状态

一句话概括：

**系统已经完成了“数据库存储 + 数据查询 + 等效计算 + 前端展示”的基础闭环。**

接下来的工作重点，不再是“有没有这个模块”，而是：

- 模块是否更稳定
- 数据是否更完整
- 架构是否更清晰
- 是否适合演示、答辩和最终交付

## 目录说明

- `back_con`：主后端
- `front_con`：后台管理前端
- `WeatherVisualization`：大屏展示前端
- `services/mysql`：数据库初始化 SQL、历史 SQL 归档、数据维护脚本
- `services/sql-agent`：SQL 智能体服务
- `services/equivalence-service`：等效计算服务
- `start-all.js`：一键启动前端、后端及 `services` 下 Docker 服务

## 下一步建议

建议后续继续按下面的顺序推进：

1. 完善自动补数脚本和任务调度
2. 优化 SQL 智能体的工具链与提示词
3. 升级等效逻辑，支持任意时间窗口按年切分与拼接
4. 整理部署与交付文档
