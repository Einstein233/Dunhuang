# 硬件输出格式器模块说明

## 模块功能

`output_formatter.py` 模块负责将实验方案（ExperimentPlan）转换为硬件系统可执行的 CSV 格式。

## 使用方式

### 方式 1：简单调用

```python
from core import ExperimentPlanGenerator, ExperimentPlanFormatter

# 1. 生成实验方案
generator = ExperimentPlanGenerator("data/data.csv", target_days=10)
plan = generator.generate()

# 2. 转换为硬件 CSV
formatter = ExperimentPlanFormatter()
formatter.to_csv(formatter.format(plan), "hardware_plan.csv")
```

### 方式 2：使用便捷函数

```python
from core import ExperimentPlanGenerator, generate_hardware_csv

# 生成实验方案并直接输出 CSV
generator = ExperimentPlanGenerator("data/data.csv", target_days=10)
plan = generator.generate()
generate_hardware_csv(plan, "hardware_plan.csv")
```

### 方式 3：自定义配置

```python
from core import ExperimentPlanGenerator, ExperimentPlanFormatter

generator = ExperimentPlanGenerator("data/data.csv", target_days=10)
plan = generator.generate()

# 自定义格式器参数
formatter = ExperimentPlanFormatter(
    default_humidity=50.0,        # 默认湿度 50%
    solar_angle_start=90.0,       # 日照起始角度 90°
    solar_angle_end=90.0,         # 日照终点角度 90°
    solar_mode='开环'             # 日照控制方式
)
formatter.to_csv(formatter.format(plan), "hardware_plan.csv")
```

## 映射规则

### 实验模式到硬件控制的映射

| 实验模式 | 舱体分配 | 温度 | 降雨 | 风速 | 日照 |
|----------|----------|------|------|------|------|
| Rain | 风雨舱 | target_temp | ON | ON | OFF |
| Rain_Pause | 风雨舱 | target_temp | OFF | ON | OFF |
| Solar | 夏季舱 | target_temp | OFF | OFF | ON |
| Dry_Baseline | 风雨舱 | target_temp | OFF | OFF | OFF |

### 详细说明

1. **Rain (脉冲喷淋阶段)**
   - 在风雨舱执行
   - 温度设置为 target_temp（降雨小时内温度中位数）
   - 降雨控制 ON，强度为 JSON 中的 intensity
   - 风速控制 ON，强度为 JSON 中的 wind_speed
   - 日照控制 OFF

2. **Rain_Pause (脉冲入渗间歇阶段)**
   - 在风雨舱执行
   - 温度设置为 target_temp
   - 降雨控制 OFF（停止喷水，让水分下渗）
   - 风速控制 ON（保持风吹，模拟雨停后的空气流动）
   - 日照控制 OFF

3. **Solar (高热日照阶段)**
   - 在夏季舱执行
   - 温度设置为 target_temp（白天温度中位数）
   - 降雨控制 OFF
   - 风速控制 OFF（防止与夏季舱冲突，积聚热量）
   - 日照控制 ON，强度为 JSON 中的 intensity_watt

4. **Dry_Baseline (夜晚/基准平衡阶段)**
   - 在风雨舱执行
   - 温度设置为 target_temp（全时段温度中位数）
   - 所有控制 OFF（静风，依靠分子扩散进行内部水分平衡）

## 输出 CSV 格式

### 字段列表（共 23 列）

1. 步骤编号
2. 工作状态（ON/OFF）
3. 运行时间（min）
4. 起始温度（℃）
5. 终点温度（℃）
6. 湿度控制标志（ON/OFF）
7. 起始湿度（%）
8. 终点湿度（%）
9. 日照控制标志（ON/OFF）
10. 日照控制方式（开环/闭环）
11. 日照起始角度（°）
12. 日照终点角度（°）
13. 起始日照强度（W/m2）
14. 终点日照强度（W/m2）
15. 降雨控制标志（ON/OFF）
16. 起始降雨量（mm/h）
17. 终点降雨量（mm/h）
18. 风速控制标志（ON/OFF）
19. 起始风速（m/s）
20. 终点风速（m/s）
21. 降雪控制标志（ON/OFF）
22. 起始降雪量（mm/h）
23. 终点降雪量（mm/h）

## 核心类说明

### ExperimentPlanFormatter

格式器主类，提供以下方法：

- `format(plan)`: 将 ExperimentPlan 转换为 HardwareStep 列表
- `to_csv(steps, output_path)`: 将 HardwareStep 列表保存为 CSV 文件
- `format_to_dict(steps)`: 将 HardwareStep 列表转换为字典列表

### HardwareStep

数据类，表示硬件执行步骤，包含所有 23 个字段。

## 约束规则

根据需求分析，系统遵循以下约束：

1. **舱体互斥**: 样本只能在一个舱中，不同舱的因素不能同时作用
2. **风速限制**: 风速控制只能发生在风雨舱，只能同步发生在降雨阶段
3. **温度控制**: 任何时刻都要控制温度，不能让样品处于非设定温度
4. **舱体切换**: 在同一时间不要生成不同舱体之间的控制参数变换
