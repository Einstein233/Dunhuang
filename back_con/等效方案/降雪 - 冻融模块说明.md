# 降雪 - 冻融循环等效模块（物理厚度直投版）

## 模块功能

本模块实现将自然降雪和冻融循环数据映射为实验室可执行的等效实验方案。

**核心特性**：采用**物理厚度直投映射**策略，直接统计降雪深度并分配到每次实验循环，适用于配备物理雪花机的实验室环境。

## 等效原理 - 物理厚度直投映射

### 核心逻辑

```
┌─────────────────────────────────────────────────────────────┐
│                    物理厚度直投映射流程                      │
├─────────────────────────────────────────────────────────────┤
│  1. 统计全年降雪总深度 (mm)                                   │
│     total_snow_depth = Σ snowfall[i]  (直接累加)            │
│                                                             │
│  2. 识别冻融循环次数 N                                        │
│     N = count(冻结→融化→再冻结 事件)                          │
│                                                             │
│  3. 计算单次实验降雪厚度                                      │
│     snow_depth_per_cycle = total_snow_depth ÷ N            │
│                                                             │
│  4. 计算硬件降雪强度 (mm/h)                                   │
│     snow_intensity = snow_depth_per_cycle ÷ (t/60)         │
│     其中 t = 降雪阶段时长 (分钟)，默认 60 min                │
└─────────────────────────────────────────────────────────────┘
```

### 与旧版本的核心区别

| 项目 | 旧版本（雪水当量） | 新版本（物理厚度直投） |
|------|-------------------|----------------------|
| 转换逻辑 | 10mm 雪 → 1mm 水 | 直接使用雪深，无转换 |
| 参数 | `snow_water_ratio = 10.0` | 已移除 |
| 计算 | `water = snow / 10.0` | `depth = snow` (直接累加) |
| 适用场景 | 喷淋模拟 | 物理雪花机 |

## 实验流程

每次冻融循环包含三个阶段：

```
┌─────────────────────────────────────────────────────────────┐
│                    一次完整冻融循环                          │
├─────────────┬──────────────┬──────────────┐                │
│  阶段 1      │   阶段 2      │   阶段 3      │                │
│  降雪阶段    │   冻结阶段    │   融化阶段    │                │
│  (Snow)     │  (Freeze)    │  (Thaw)      │                │
│  风雪舱      │   风雪舱      │   温度升高    │                │
│  雪花机 ON   │   雪花机 OFF  │   雪花机 OFF  │                │
│  温度 -10~-2℃│  温度 Tmin   │  温度 Tmax   │                │
│  日照 OFF    │   日照 OFF    │   日照 OFF    │                │
│  降雨 OFF    │   降雨 OFF    │   降雨 OFF    │                │
└─────────────┴──────────────┴──────────────┘                │
└─────────────────────────────────────────────────────────────┘
```

### 各阶段详细说明

| 阶段 | 舱体 | 温度范围 | 降雪 | 日照 | 降雨 | 时长 |
|------|------|----------|------|------|------|------|
| Snow | 风雪舱 | -10℃ ~ -2℃ | ON | OFF | OFF | 60 min (可调) |
| Freeze | 风雪舱 | Tmin (-18℃) | OFF | OFF | OFF | 2.5 h |
| Thaw | 风雪舱 | Tmax (5℃) | OFF | OFF | OFF | 1.5 h |

## 快速开始

### 方式 1：使用便捷函数

```python
from core import generate_snow_ftc_plan

# 生成实验方案
plan = generate_snow_ftc_plan(
    csv_path="data/processed_weather_data.csv",
    target_days=10.0,
    output_json="snow_ftc_plan.json",
    output_csv="snow_ftc_hardware_plan.csv"
)
```

### 方式 2：分步调用

```python
from core import SnowFTCPlanGenerator, SnowFTCFormatter

# 1. 创建生成器
generator = SnowFTCPlanGenerator(
    csv_path="data/processed_weather_data.csv",
    target_days=10.0
)

# 2. 生成实验方案
plan = generator.generate()

# 3. 格式化为硬件 CSV
formatter = SnowFTCFormatter()
steps = formatter.format(plan)
formatter.to_csv(steps, "hardware_plan.csv")
```

### 方式 3：单独使用各组件

```python
from core import (
    WeatherDataLoader,
    SnowEventExtractor,
    FTCEventIdentifier,
    SnowDepthMapper
)

# 加载数据
loader = WeatherDataLoader("data/data.csv")
dataset, stats = loader.load_with_statistics()

# 提取降雪事件（直接统计雪深）
snow_extractor = SnowEventExtractor(snow_temp_threshold=0.0)
snow_events = snow_extractor.extract(dataset)
snow_stats = snow_extractor.get_statistics(snow_events)

print(f"降雪事件数：{snow_stats['total_snow_events']}")
print(f"总降雪深度：{snow_stats['total_snow_depth_mm']} mm")

# 识别冻融循环
ftc_identifier = FTCEventIdentifier(
    freeze_start=-2.2,
    thaw_start=1.2
)
ftc_events = ftc_identifier.identify(dataset)
print(f"冻融循环数：{len(ftc_events)}")

# 计算单次降雪厚度
mapper = SnowDepthMapper()
n_cycles = len(ftc_events)
snow_per_cycle = mapper.distribute_snow_depth(
    snow_stats['total_snow_depth_mm'],
    n_cycles
)
print(f"单次降雪厚度：{snow_per_cycle:.2f} mm")
```

## 配置参数

### 默认配置 (DEFAULT_SNOW_FTC_CONFIG)

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `freeze_start` | -2.2 | 冻结启动阈值 (°C) |
| `thaw_start` | 1.2 | 融化启动阈值 (°C) |
| `snow_temp_threshold` | 0.0 | 降雪温度阈值 (°C) |
| `lab_Tmin` | -18.0 | 实验室最低温 (°C) |
| `lab_Tmax` | 5.0 | 实验室最高温 (°C) |
| `lab_freeze_hours` | 2.5 | 冻结时长 (小时) |
| `lab_thaw_hours` | 1.5 | 融化时长 (小时) |
| `snow_duration_min` | 60.0 | **单次降雪阶段时长 (分钟)** |
| `snow_temp_min` | -10.0 | **降雪阶段最低温 (°C)** |
| `snow_temp_max` | -2.0 | **降雪阶段最高温 (°C)** |

### 自定义配置

```python
custom_config = {
```

### CSV 格式 (snow_ftc_hardware_plan.csv)

硬件可执行的 23 列 CSV 文件。降雪阶段的控制标志设置：

```csv
步骤编号，工作状态，运行时间，起始温度，终点温度，...,日照控制，降雨控制，降雪控制，起始降雪量，终点降雪量

4,ON,150.0,-18.0,-18.0,...,OFF,OFF,OFF,0.0,0.0   <- 冻结阶段
...
```

**降雪阶段控制规则**：
- `降雪控制标志` = ON
- `日照控制标志` = OFF（防止雪花融化）
- `降雨控制标志` = OFF（使用雪花机而非喷淋）
- `起始降雪量` = `终点降雪量` = 计算的硬件降雪强度 (mm/h)

## 核心公式

### 降雪强度计算

```
单次降雪深度 (mm) = 全年总雪深 (mm) ÷ 冻融循环次数 N

硬件降雪强度 (mm/h) = 单次降雪深度 (mm) ÷ (降雪时长 (min) ÷ 60)
```

### 示例计算

假设：
- 全年总雪深 = 50 mm
- 冻融循环次数 = 25 次
- 降雪阶段时长 = 60 min

则：
```
单次降雪深度 = 50 ÷ 25 = 2.0 mm
硬件降雪强度 = 2.0 ÷ (60/60) = 2.0 mm/h
```

若降雪时长改为 30 min：
```
硬件降雪强度 = 2.0 ÷ (30/60) = 4.0 mm/h
```

## 注意事项

1. **数据要求**：输入 CSV 必须包含 `snowfall` 列（降雪量，mm/h）

2. **降雪温度范围**：降雪阶段温度自动限制在 -10℃ ~ -2℃之间，以保证雪花物理形态

3. **日照互斥**：降雪阶段日照控制强制为 OFF，防止雪花融化

4. **循环次数**：由数据中识别到的冻融事件数量决定，不是固定值

5. **雪花机校准**：硬件降雪强度 (mm/h) 需要根据实际雪花机输出特性进行校准
