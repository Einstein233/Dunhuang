# 🚀 敦煌数据智能体 (Dunhuang Data Agent) V2.0

## 📖 项目概述
本项目是一个基于大语言模型（LLM）与微服务架构的商业智能数据分析系统。系统能够通过自然语言交互，自动执行 Text-to-SQL，并动态渲染 ECharts 可视化图表，同时提供思维链路追踪与高阶排障能力。

**核心技术栈：**
- **前端 (展示层):** RuoYi Vue (Vue 2/3 + Element UI) | 运行环境: Node 16
- **后端 (大脑层):** Node.js + Express + Prisma ORM | 运行环境: Node 24
- **数据底座:** MySQL 8.0 (基于 Docker 容器化与 Volume 持久化)

---

## 🛠️ 环境准备与依赖
在部署本项目前，请确保目标服务器/本地电脑已安装以下底层依赖：
1. **Docker & Docker Compose** (提供数据库与生产环境隔离)
2. **NVM (Node Version Manager)** (用于快速切换前后端所需的 Node 版本)

# 部署模式一：本地极速开发流 (推荐开发者使用)

此模式将数据库放入 Docker 后台静默运行，后端代码在宿主机使用 tsx watch 进行毫秒级热更新开发。
Step 1: 启动数据底座
# 后台静默启动 MySQL 容器 (映射至 3308 端口)
docker-compose up -d mysql-db

Step 2: 数据库初始化 (首次运行必需)
将 SQL 文件夹内的基础图纸注入数据库：
# 1. 将文件投递进容器
docker cp ./SQL/menu.sql dunhuang-mysql-prod:/tmp/menu.sql
docker cp ./SQL/weather_data.sql dunhuang-mysql-prod:/tmp/weather_data.sql

# 2. 进入容器执行初始化
docker exec -it dunhuang-mysql-prod mysql -u root -proot
mysql> USE dunhuang_agent;
mysql> source /tmp/menu.sql;
mysql> source /tmp/weather_data.sql;
mysql> exit;

Step 3: 启动后端大模型 Agent 服务
# 切换至高版本 Node
nvm use 24
# 安装依赖
npm install
# 以热更新模式启动服务 (监听 3000 端口)
npm run dev

Step 4: 启动前端若依系统 (需在若依前端目录下执行)
# 切换至稳定版 Node
nvm use 16
# 启动前端页面
npm run dev


## 部署模式二：生产环境封箱打包 (推荐项目交付与上云)
此模式将 Node 后端与 MySQL 数据库全部封箱，实现跨平台的一键部署。

# 一键构建并启动整个微服务集群
docker-compose up -d --build
注：生产模式下，MySQL 容器会通过挂载卷 ./mysql_data (或内部 Volume) 持久化数据，即使容器被销毁，数据依然安全。

常见维护指令 (Cheat Sheet)
更新数据库字段结构： 修改 schema.prisma 后，执行 npx prisma db push，即可自动完成表结构的热更新。

查看大模型后台日志： docker logs -f dunhuang-agent-prod

清理闲置的容器与网络： docker-compose down
