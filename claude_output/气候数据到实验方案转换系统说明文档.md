# 气候数据到实验方案转换系统说明文档

## 1. 系统概述

### 1.1 系统目标

本系统实现从**长期气候数据**（如 10 年逐小时气象数据）到**实验室等效实验方案**的自动转换。核心目标是：

- **机理可解释性**：每个转换步骤都基于物理/气候学原理
- **全国适配性**：算法参数化设计，支持全国各地不同气候区
- **闭环输出**：直接生成可执行的实验室实验方案

### 1.2 核心环境因子

系统映射以下环境因子到实验室条件：

| 环境因子 | 自然场景 | 实验室等效 |
|---------|---------|-----------|
| 温度 | 逐小时气温 | 季节等效温度 (P25/P50/P75 分位数) |
| 降雨 | 逐小时降雨量 | 等效降雨事件 (次数、时长、强度) |
| 日照 | 短波辐射 | 雨后窗口等效辐照度 (P90 分位数) |
| 湿度 | 相对湿度 | 季节平均相对湿度 |
| 风速 | 10m 风速 | 季节平均风速 |

## 3. 核心算法详解

### 3.1 季节分割算法 (`Ssegment.py`)

#### 3.1.1 冬季识别 (`winter` 函数)

**输入**：逐日最低温序列 `tmin[]`

**参数**：
- `T0`：冷阈值（默认 0°C）
- `g`：暖间隙容忍天数（默认 6 天）
- `min_len`：冬季最小长度（默认 7 天）
- `post_cold_min`：回暖后重新变冷的最小持续天数（默认 5 天）

**算法步骤**：
1. 标记冷日：`cold[t] = 1 if tmin[t] < T0 else 0`
2. 构建连续段：允许 `cold=0` 的连续长度 ≤ `g` 不断开
3. 过滤短段：剔除长度 < `min_len` 的段
4. 桥接处理：若回暖期 ≤ `min_len` 且之后重新变冷 ≥ `post_cold_min` 天，则桥接为一个冬季段

**输出**：
```python
{
    "W_idx": [(l1, r1), (l2, r2), ...],  # 冬季索引区间
    "W_date": [(date_l1, date_r1), ...],  # 冬季日期范围
    "K_w": 冬季段数量
}
```

#### 3.1.2 非冬季三段切分 (`split` 函数)

**输入**：非冬季区间的逐日平均温序列 `tmean[p:q]`

**参数**：
- `min_seg_len`：最小段长度（默认 15 天）
- `smooth_win`：温度平滑窗口（默认 7 天）
- `run_need`：连续判定天数（默认 10 天）
- `delta`：温度裕度（默认 6.5°C）

**算法步骤**：
1. 平滑温度序列（移动平均）
2. 找到峰值点（强制夏季包含最高温）
3. **τ1（春/夏分界）**：
   - 计算春季参考温度（前 `min_seg_len` 天的中位数）
   - 阈值 = 参考温度 + `delta`
   - 找首次连续 `run_need` 天 ≥ 阈值的起点 → 夏季开始
4. **τ2（夏/秋分界）**：
   - 计算夏季核心参考温度（[τ1+1, peak] 的中位数）
   - 阈值 = 参考温度 - `delta`
   - 找首次连续 `run_need` 天 ≤ 阈值的起点 → 秋季开始

**输出**：
```python
(spring, summer, autumn)  # 三元组，每个为 (l, r) 索引区间或 None
```

### 3.2 降雨等效算法 (`exp.py`)

#### 3.2.1 降雨事件提取

**参数**：
- `min_raingap`：事件合并间隔（默认 3 小时）
- `min_rainfall`：最小事件雨量（默认 0.3mm）

**步骤**：
1. 识别连续降雨（间隔 ≤ `min_raingap` 视为同一事件）
2. 累加事件总雨量
3. 过滤小于 `min_rainfall` 的事件

#### 3.2.2 阈值聚类 (`rainfall_cluster_thresholds`)

**目的**：确定当地小/中/大雨的自然分界阈值

**方法**：
1. 取所有有雨小时（`rain > 0`）
2. 对数变换：`x = log1p(rain)`
3. GMM 聚类（3 类）
4. 阈值计算：
   - `T1 = exp((mean1 + mean2) / 2) - 1`（小/中雨分界）
   - `T2 = exp((mean2 + mean3) / 2) - 1`（中/大雨分界）

#### 3.2.3 事件聚类与推荐

**步骤**：
1. GMM 二分类（小事件/大事件）
2. 推荐事件雨量 = `max(large_events.mean(), P75)`

#### 3.2.4 实验方案计算

**输出结构**（每工况）：
```json
{
  "natural_stats": {
    "avg_intensity": 1.23,      // 自然平均雨强 (mm/h)
    "raw_total": 45.6           // 自然总雨量 (mm)
  },
  "lab_settings": {
    "intensity_setting": 20,    // 实验室雨强 (mm/h)
    "total_duration_min": 137.5 // 实验室总时长 (分钟)
  },
  "execution_plan": {
    "event_count": 5,           // 实验次数
    "single_duration_min": 27.5 // 单次时长 (分钟)
  }
}
```

### 3.3 日照等效算法 (`solar_equivalence.py`)

#### 3.3.1 雨后窗口定义

**参数**：
- `wmax_hours`：最大窗口长度（默认 48 小时）

**窗口计算**：
- `window_start = rain_end + 1h`
- `window_end = min(rain_end + wmax_hours, next_rain_start - 1h)`

#### 3.3.2 Ecap 计算

**公式**：
```
Ecap_i = Σ(shortwave_radiation[t] * 3600 / 1e6)  for t in window
单位：MJ/m²
```

#### 3.3.3 代表强度计算

**步骤**：
1. 收集所有雨后窗口的短波辐射样本
2. 过滤日光样本（`> daylight_threshold`）
3. 取 P90 分位数作为代表强度 `I_rep`

---

### 3.4 环境因子等效 (`seasonal_equivalence_ssegment.py`)

#### 3.4.1 温度等效规则

| 季节 | 分位数 | 物理意义 |
|-----|--------|---------|
| 春 | P50 | 典型温度 |
| 夏 | P75 | 偏高温度（加速老化） |
| 秋 | P50 | 典型温度 |
| 冬 | P25 | 偏低温度 |

#### 3.4.2 湿度/风速等效

- 相对湿度：季节均值
- 风速：季节均值

## 6. 使用示例

### 6.1 命令行运行

```bash
python full_experiment_pipeline_v2.py \
  --csv back_con/data/processed_weather_data.csv \
  --wmax 48 \
  --p 90 \
  --T0 0.0 \
  --g-days 6 \
  --out full_pipeline_result_v2.json
```

### 6.2 Python API

```python
from full_experiment_pipeline_v2 import run_full_pipeline

result = run_full_pipeline(
    csv_path="back_con/data/processed_weather_data.csv",
    wmax_hours=48,
    intensity_p=90.0,
    T0=0.0,
    g_days=6
)

# 访问春季实验方案
spring_rain = result["per_season"]["spring"]["rainfall_equivalence"]
spring_env = result["per_season"]["spring"]["env_equivalence"]
```

## 8. 设计特点

### 8.1 可解释性
- 季节分割基于明确的温度阈值和物理规则
- 等效参数使用分位数而非黑箱模型
- 每一步输出可审计

### 8.2 灵活性
- 所有关键参数可配置
- 支持跨年冬季（年初 + 年末两段）
- 适配不同气候区（通过调整 `T0`、`g` 等）

### 8.3 鲁棒性
- 少雨/无雨场景的 fallback 处理
- 空数据场景的安全返回
- JSON 序列化保护（处理日期/NumPy 类型）
