# 敦煌项目一键启动脚本使用说明

## 概述

本项目包含三个子服务，需要分别启动：
- **WeatherVisualization**: 前端可视化服务
- **front_con**: 前端管理服务  
- **back_con**: 后端API服务

为了简化启动流程，我创建了一个统一启动脚本 `start-all.js`，可以一键启动所有服务。

## 使用方法

### 1. 激活 conda 环境

首先确保已激活 dunhuang 环境：

```bash
conda activate dunhuang
```

### 2. 一键启动所有服务

在项目根目录（DH_web）下运行：

```bash
node start-all.js
```

### 3. 停止所有服务

在运行的终端中按 `Ctrl + C`，即可停止所有服务。

## 服务说明

启动后，脚本会在终端中显示三个服务的实时日志，并使用不同颜色区分：

- 🔵 **[WeatherVisualization]** - 青色显示日志
- 🟢 **[front_con]** - 绿色显示日志
- 🟡 **[back_con]** - 黄色显示日志

## 技术实现

该脚本使用 Node.js 的 `child_process.spawn` 方法并行启动三个 npm 进程，并：
- 统一管理所有服务的输出和错误信息
- 使用颜色区分不同服务的日志
- 监听 Ctrl+C 信号，一键停止所有服务

## 优势

相比手动启动三个服务，使用该脚本的优势：

1. **一键启动**：无需切换目录多次执行命令
2. **统一管理**：所有服务的日志集中显示
3. **快速停止**：Ctrl+C 同时停止所有服务
4. **清晰区分**：通过颜色标签区分不同服务

## 注意事项

- 确保已安装 Node.js 和 npm
- 确保三个子项目的依赖已安装（`npm install`）
- 确保端口未被占用
- 使用前请激活 dunhuang conda 环境

## 故障排查

如果某个服务启动失败：

1. 检查对应子项目的依赖是否完整安装
2. 检查端口是否被其他程序占用
3. 查看终端中对应颜色的错误日志信息
4. 可以单独进入对应子目录手动运行 `npm run dev` 查看详细错误
